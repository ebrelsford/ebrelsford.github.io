Part 1: CARTO & CartoCSS

Create a map in CARTO. Then do the following:

  1. Add at least one zoom-specific styling rule. That is, your features' styles should change somehow according to the map's zoom level.
  2. Add at least one attribute-specific styling rule. That is, some features should be styled differently according to the value of one or more of their attributes.
  3. Add a property that your styles are not currently using. You can find all of the properties in the CartoCSS documentation. Mostly points will use properties that start with marker-, polygons will use properties that start with polygon-.
  4. Consider how the map will look when shared online. Give thought to the title, description, and pop-ups you use. Add text boxes as it makes sense to, including context to aid the reader. Modify your pop-up's labels to make them more readable.

Submit a link to the map (please leave it online at least until I give you a grade for the assignment), and the final CartoCSS (copy and paste into a document and create a PDF).

Part 2: CartoCSS

Here's a little bit of CartoCSS from OpenStreetMap's Humanitarian layer:

@power_line: #7a5960;

#power-line[zoom>=16] {
    line-color: @power_line;
    line-width: 1;
}
#power-point[type='pole'][zoom>=18] {
    marker-fill: @land;
    marker-width: 4;
    marker-height: 4;
    marker-line-color: @power_line;
    marker-line-width: 2;
}

Answer the following:

  1. When will the second statement (the one starting with #power-point) apply? That is, does it work at all zoom levels? Which features will be styled this way?
  2. What color(s) will the power-line and power-point layers be styled with? You don't have to describe the color itself (eg "red" or "blueish"), but you should write the color the way you would in CartoCSS.
